# Heuristic Baselines

Baseline solutions for the ACCIDENT competition: detecting **when**, **where**, and **what type** of traffic accident occurs in video footage.

## Baselines

| Baseline          | Temporal                             | Spatial                      | Type                      | Key idea                          |
|-------------------|--------------------------------------|------------------------------|---------------------------|-----------------------------------|
| **Naive**         | Video midpoint                       | Frame center                 | Majority class (`single`) | No computation, just defaults     |
| **Optical flow**  | Change-point on motion magnitude     | Frame center                 | Majority class (`single`) | Pixel motion spikes at collision  |
| **Bbox dynamics** | Change-point on total bbox area      | Closest object pair midpoint | Majority class (`single`) | Object sizes change at collision  |
| **Ensemble**      | Mean of optical flow + bbox dynamics | From bbox dynamics           | Majority class (`single`) | Averages two temporal predictions |

## Structure

**Standalone scripts** (`.py`) — run the full pipeline end-to-end and print the evaluation metrics as reported in the paper:
- `naive.py` — naive baseline
- `optical_flow.py` — optical flow baseline
- `bbox_dynamics.py` — bounding box dynamics baseline

**Notebooks** (`.ipynb`) — equivalent code, but with detailed documentation explaining the methods, step-by-step markdown cells for understanding, and bonus visualizations:
- `naive.ipynb`, `optical_flow.ipynb`, `bbox_dynamics.ipynb`, `ensemble.ipynb`

**Shared intermediate results** (gitignored, generated by the scripts/notebooks):
- `optical_flow.pkl` — precomputed optical flow scores for all videos
- `inference-yolo11x/` — per-video YOLO detection JSONs
- `output_*.csv` — prediction CSVs (used for ensemble and cross-baseline evaluation)

**Shared utilities:**
- `metrics.py` — evaluation metrics (temporal and spatial accuracy)

## Setup

```bash
uv sync
```

The dataset is expected at `../../dataset` relative to this directory by default, but every script also supports `--dataset-path`.

Expected layout:

```text
dataset/
  metadata-real.csv
  real_videos/
    ...
```

## Running

All scripts resolve paths relative to their own location, so they work from any working directory.

```bash
# Naive baseline — instant, no GPU needed
python baselines/heuristic/naive.py

# Optical flow — CPU-heavy, parallelized with joblib
python baselines/heuristic/optical_flow.py              # all videos
python baselines/heuristic/optical_flow.py --take 5     # quick test

# Bbox dynamics — GPU-heavy (YOLO inference)
python baselines/heuristic/bbox_dynamics.py             # all videos
python baselines/heuristic/bbox_dynamics.py --take 5    # quick test
```

Each script prints evaluation metrics at the end. If the other baseline's output CSV is available, ensemble results are printed as well.

Use `--help` on any script for full argument documentation.

## Recommended first run

If you are validating a fresh environment, use this order:

1. `python baselines/heuristic/naive.py`
2. `python baselines/heuristic/optical_flow.py --take 5`
3. `python baselines/heuristic/bbox_dynamics.py --take 5`

That sequence checks dataset paths first, then CPU processing, then the heavier GPU-dependent baseline.
